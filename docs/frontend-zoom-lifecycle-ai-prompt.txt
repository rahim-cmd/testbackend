Relevant endpoints:
1) GET /bookings/my

5) GET /circles/admin

1) One Zoom meeting belongs to one circle.

3) If admin updates circle schedule, backend syncs Zoom meeting and all approved booking snapshots.
4) If Zoom link changes or is removed, backend updates user dashboard state and sends email.
5) If meeting time is over, backend marks link expired in dashboard response.

Dashboard fields from GET /bookings/my that frontend must use:
- booking_status

- approved_at
- zoom_meeting_id
- zoom_link
- zoom_start_time

- zoom_updated_at
- zoom_status
- zoom_message

Possible zoom_status values:
- pending
- active
- updated
- unavailable
- expired
- rejected
- cancelled

Required user dashboard behavior:
1) If zoom_status = active, show Join Meeting button.
2) If zoom_status = updated, show Join Meeting button and a visible Updated badge/message.
3) If zoom_status = unavailable, hide join button and show warning message from zoom_message.
4) If zoom_status = expired, hide join button and show Meeting Completed or Link Expired state.
5) If booking_status = pending, show waiting state.
6) If booking_status = rejected, show rejection state and reason if notes/zoom_message available.
